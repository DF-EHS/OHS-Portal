# 大豐環保科技 — 職業安全衛生入口網站（OHS Portal）系統導覽文件

**版本：** 2.2　　**日期：** 2026-07-02　　**維護：** 賴佑毓 / 人力資源部

---

## 一、系統概述

大豐環保科技職業安全衛生入口網站（OHS Portal）是整合公司各項職安衛管理功能的內部數位平台，目前上線功能涵蓋 **20 個系統模組**（另有 1 個建置中）、**5 套互動教材**及 3 個 AI 快捷工具，供職安管理人員、現場主管及全體員工日常使用。

系統採靜態前端架構部署於 GitHub Pages，核心 AI 功能透過公司 IT 部門 API Gateway 串接 Gemini 2.5 Flash 模型，兼顧資安管控與使用便利；部分系統以 Google Apps Script + Google Sheets 作為後端資料儲存，無需另建伺服器。

## 二、系統架構

### 2.1 技術架構

| 層次 | 元件 | 說明 |
|------|------|------|
| 前端展示層 | GitHub Pages | 靜態 HTML/CSS/JS，所有使用者透過瀏覽器存取，無需安裝 |
| 資料儲存層 | Google Apps Script + Google Sheets | 處理 CRUD 操作，各系統各自獨立一個 Spreadsheet |
| 中介轉發層 | Cloudflare Worker | 解決瀏覽器 CORS 限制、轉發 AI 請求，金鑰不暴露於前端 |

職安動態新聞由 GitHub Actions 每日自動抓取並更新靜態頁面；部分系統另有 AI 功能透過 Cloudflare Worker 串接。

### 3.1 承攬商進場申請——QR Code 掃碼填報流程

**情境說明：** 承攬商抵達廠區後，以手機掃描張貼於警衛室或由管理員事先提供的廠商專屬 QR Code，系統自動帶入廠商資料，引導完成五步驟進場申請表單，全程不需要帳號或密碼。

**參與角色：** 承攬商現場人員、GAS 後端（廠商名冊 / 進場申請）、HR 管理員

```mermaid
sequenceDiagram
    actor V as 承攬商人員
    participant QR as QR Code
    participant SPA as contract/index.html
    participant GAS as Google Apps Script
    actor HR as HR 管理員

    Note over QR: 廠商專屬：?mode=fill&v=VDR-001<br/>通用掃碼：?mode=fill

    V->>QR: 掃描 QR Code
    QR->>SPA: 帶 URL 參數跳轉
    SPA->>SPA: initFromUrl() 執行<br/>隱藏側邊欄，設定 guest 角色
    SPA->>GAS: GET listVendors
    GAS-->>SPA: 廠商名冊 JSON

    alt 廠商專屬 QR Code（含 v=VDR-xxx）
        SPA->>SPA: 比對代碼，預填<br/>公司名稱 / 聯絡人 / 電話<br/>欄位以淺藍底色標示「已自動帶入」
    else 通用 QR Code（無 v 參數）
        SPA->>V: 顯示申請類型選單<br/>（進場申請 / 動火 / 特殊危害）
    end

    V->>SPA: 選擇申請類型，進入填報表單
    Note over V,SPA: Step 1 基本資料<br/>Step 2 危害告知確認<br/>Step 3 安全承諾書簽署<br/>Step 4 作業人員清單<br/>Step 5 確認送出

    V->>SPA: 完成五步驟，點擊送出
    SPA->>GAS: POST submit {完整申請資料 JSON}
    GAS->>GAS: 新增一列至「進場申請」Sheet<br/>col1~14 攤平欄位，col15 存完整 JSON
    GAS-->>SPA: {success: true, id: "REC-xxxxxxxx"}
    SPA->>V: 顯示完成畫面，告知記錄編號

    Note over HR,GAS: 事後管理作業

    HR->>SPA: 以管理員帳號登入後台
    SPA->>GAS: GET list
    GAS-->>SPA: 所有進場申請摘要
    HR->>SPA: 點擊單筆 → 查看完整申請內容

    alt 動火 / 特殊危害作業申請
        HR->>SPA: 填寫巡檢記錄與主管核可
        SPA->>GAS: POST update {id, 巡檢資料}
    end

    HR->>SPA: 作業完工後填寫績效評核
    SPA->>GAS: POST saveEval {id, 評核資料}
    GAS->>GAS: 寫入「evaluations」Sheet
    GAS-->>SPA: {ok: true}
```

### 3.2 AI 現場危害辨識流程

**情境說明：** 職安人員或現場主管在巡檢時，以手機拍攝現場照片上傳至危害鑑別系統，由 AI 自動識別畫面中的危害因子，給出風險等級評估與具體改善建議，整個過程約 10–20 秒。

**參與角色：** 職安人員、Cloudflare Worker（AI 代理）、IT API Gateway、Gemini 2.5 Flash Vision

```mermaid
sequenceDiagram
    actor U as 職安人員
    participant SPA as risk/index.html
    participant CW as Cloudflare Worker
    participant GW as IT API Gateway
    participant AI as Gemini 2.5 Flash

    U->>SPA: 上傳現場照片（JPG/PNG）
    SPA->>SPA: 讀取 File，轉換為 Base64 字串
    SPA->>SPA: 組裝請求 Payload<br/>{model, messages:[{role:user,<br/>content:[{type:image_url, url:base64},<br/>{type:text, text:危害辨識提示詞}]}],<br/>stream:true}

    SPA->>CW: POST / （Payload）
    Note over CW: CORS Proxy 層<br/>注入 Authorization: Bearer ${env.API_KEY}

    CW->>GW: POST /api/v1/model/chat
    GW->>AI: Vision 推論請求
    Note over AI: 辨識畫面中的：<br/>- 缺乏防護設備<br/>- 不安全行為<br/>- 環境危害<br/>- 違規情形

    loop 串流回應（SSE）
        AI-->>GW: 逐 token 輸出
        GW-->>CW: text/event-stream
        CW-->>SPA: 轉發 SSE 事件
        SPA->>U: 即時渲染回應文字
    end

    SPA->>U: 完整呈現：<br/>① 危害清單與風險等級<br/>② 法規依據條文<br/>③ 建議改善措施<br/>④ 優先處理順序
    U->>SPA: 可複製結果 / 截圖存檔
```

### 3.3 職安新聞自動更新管線

**情境說明：** 系統每日自動從職安署 RSS 與 Google News 抓取最新職安相關新聞，透過 AI 生成繁體中文摘要，無需人工介入，即可維持新聞頁面每日更新。每週一另自動產生週報草稿供職安人員審核後發送。

**參與角色：** GitHub Actions 排程器、Python 腳本、外部新聞來源、Gemini AI、GitHub Repository

```mermaid
sequenceDiagram
    participant Cron as GitHub Actions<br/>Cron: 0 1 * * * (UTC)
    participant PY as fetch_news.py
    participant RSS as 職安署 RSS Feed
    participant GN as Google News API
    participant Cache as news/cache.json
    participant CW as Cloudflare Worker
    participant AI as Gemini 2.5 Flash
    participant GH as GitHub Repository

    Note over Cron: 每日 09:00 台灣時間自動觸發<br/>亦可由 workflow_dispatch 手動執行

    Cron->>PY: 啟動執行
    PY->>RSS: GET 職安署公告 Feed
    RSS-->>PY: Atom/RSS 條目清單
    PY->>GN: GET Google News（關鍵字：職業災害、職安法）
    GN-->>PY: 新聞條目清單

    PY->>Cache: 讀取現有快取
    Cache-->>PY: 過去 14 天已處理條目
    PY->>PY: 去重：過濾已存在 URL
    PY->>PY: 篩選：僅保留相關性高的條目

    loop 每篇新增新聞
        PY->>CW: POST {文章標題 + 內文}
        CW->>AI: 摘要生成請求（繁體中文，100 字內）
        AI-->>CW: 摘要文字
        CW-->>PY: 摘要結果
    end

    PY->>Cache: 更新 cache.json（移除 14 天前舊條目）
    PY->>GH: 產生 news/index.html（含所有摘要）
    GH->>GH: git commit -m "auto: update news [skip ci]"
    GH->>GH: git push → GitHub Pages 自動部署

    Note over GH: [skip ci] 標記防止<br/>觸發 Actions 循環執行

    Note over PY,GH: 每週一另執行週報流程：<br/>彙整上週新聞 → AI 生成週報 →<br/>開啟 Outlook 草稿 → 人工確認後發送
```

### 3.4 人因性危害預防——完整年度作業週期

**情境說明：** 依職安法第 6 條第 2 項，雇主每年須對可能患有肌肉骨骼疾病的員工實施症狀調查、評估並採取改善措施。本流程從年度問卷派送，到改善措施完成追蹤，覆蓋完整的 PDCA 循環。

**參與角色：** 職安人員（觸發）、員工（填報）、AI（初步分析）、護理師/醫師（評估）、HR（追蹤）

```mermaid
flowchart TD
    A([年度開始\n職安人員觸發問卷]) --> B[產生填報連結\n發送至各部門]
    B --> C{員工收到通知}

    C --> D[輸入員工編號\n進入人體圖問卷]
    D --> E[各部位評分\n0~5 分 × 11 個部位\n頸/肩/上臂/肘/前臂\n手腕/手/下背/大腿/膝/小腿足踝]
    E --> F[填寫症狀持續時間\n1月 / 3月 / 6月 / 1年 / 3年以上]
    F --> G[送出問卷]
    G --> H[(GAS 寫入\n問卷回覆 Sheet)]

    H --> I{AI 自動風險分析}
    I -->|總分低 各部位 < 2| J[低風險\n建議自我保健]
    I -->|部分部位 ≥ 3| K[中風險\n建議護理師評估]
    I -->|多部位 ≥ 4 或持續 > 6月| L[高風險\n建議立即就醫]

    J & K & L --> M[HR 後台檢視\n問卷回覆清單]
    M --> N{需要面談?}
    N -->|否| O[僅記錄存檔\n列入年度統計]
    N -->|是| P[護理師 / 醫師約談]

    P --> Q[填寫面談評估意見\nnurseNote / aiNote]
    Q --> R[登錄改善措施\n工程控制 / 行政控制\n負責人 / 期限]
    R --> S[(GAS 寫入\n改善措施 Sheet)]

    S --> T{期限內完成?}
    T -->|是| U[填寫成效複評\n標記措施完成]
    T -->|否| V[系統逾期標示\n通知負責人]
    V --> R

    U --> W[合規儀表板更新\n填答率 / 高風險比率\n措施完成率]
    O --> W
    W --> X([年度報告產出\n送主管審閱])
```

### 3.5 法規合規稽核——季度自動審查流程

**情境說明：** 每季首日由 Claude Code Remote Agent 自動觸發，比對現行適用法規是否有新修訂、新發布或廢止，提示職安人員執行手動確認步驟，確保法規鑑別清單維持最新狀態，符合職安衛管理系統持續改善要求。

**參與角色：** Remote Agent（自動觸發）、職安人員（執行）、法規問答 AI、Git 版控

```mermaid
sequenceDiagram
    participant Cron as Claude Code<br/>Remote Trigger (季度 Cron)
    participant Agent as Remote Agent<br/>（Anthropic Cloud）
    actor HR as 職安人員
    participant SPA as law/index.html
    participant GAS as 法規資料庫 GAS
    participant CW as Cloudflare Worker
    participant AI as Gemini 2.5 Flash

    Note over Cron: Q1: 3/1 09:00<br/>Q2: 6/1 09:00<br/>Q3: 9/1 09:00<br/>Q4: 12/1 09:00

    Cron->>Agent: 觸發季度稽核任務
    Agent->>Agent: 讀取 law/ 現有鑑別清單
    Agent->>Agent: 比對法規修訂時間戳記
    Agent->>HR: 推送提醒通知<br/>「本季法規合規審查待執行」<br/>附上執行步驟說明

    HR->>SPA: 開啟法規鑑別查詢系統
    SPA->>GAS: GET 法規清單（含最後更新日期）
    GAS-->>SPA: 法規條目列表

    loop 逐條審查
        HR->>SPA: 點擊法規條目
        SPA->>CW: POST {法規名稱, 詢問是否有新修訂}
        CW->>AI: 查詢最新立法動態
        AI-->>CW: 修訂狀態回應
        CW-->>SPA: 顯示修訂資訊

        alt 有新修訂
            HR->>SPA: 點擊「更新鑑別結果」
            SPA->>GAS: POST update {法規編號, 新狀態, 更新日期}
            GAS-->>SPA: {success: true}
        else 無異動
            HR->>SPA: 標記「本季確認完成」
        end
    end

    HR->>SPA: 匯出鑑別清單（PDF / 列印）
    Note over HR: 存檔供稽核查核<br/>法規鑑別為 ISO 45001<br/>必要文件化資訊
```

## 四、功能模組索引

側邊欄依功能性質分為八個分區（另有小工具區），共 20 個入口（含 2 個外部系統連結）。

### 4.1 安全衛生

| 系統 | 路徑 | 主要功能 |
|------|------|---------|
| 危害鑑別及風險評估 | `risk/` | 查詢各廠區作業危害鑑別表，含風險等級評分、管制措施及不可接受風險清單 |
| 職業災害分析系統 | `accident/` | 彙整歷年職災事故資料，自動計算 FR／SR，支援趨勢分析、部門分類及事故類型統計 |
| 職災情報與安衛動態 | `news/` | 自動彙整職安署公告、重大職災新聞及法規更新，工作日每四小時由 AI 自動摘要更新 |

> 🌡️ 熱危害風險即時查詢（`heat/`）功能已整合於側邊欄小工具區，不另佔導覽項目。

### 4.2 機械設備及化學品

| 系統 | 路徑 | 主要功能 |
|------|------|---------|
| 機械設備管理系統 | `equipment/` | 設備清冊管理與安全作業標準查閱，整合 29 份 SOP 文件，涵蓋作業類、廠區設備、再生處理設備等各類機械操作標準 |
| 危害性化學品管理系統 | `chemical/` | 全興廠化學品清冊（GHS 危害分類）、SDS 安全資料表索引與線上預覽、公共危險物品及優先管理化學品申報合規追蹤 |
| 自動點檢系統 | 外部連結 | 機械設備定期點檢作業系統，支援巡查記錄數位化、點檢項目管理及異常通報（由 check-system.gm.zerozero.tw 提供） |

### 4.3 人員管理

| 系統 | 路徑 | 主要功能 |
|------|------|---------|
| 承攬商管理系統 | `contract/` | 承攬商進場申請（危害告知確認、安全承諾書、工作許可證）、動火作業及特殊危害作業申請全程電子化；支援 QR Code 掃碼填報（廠商專屬 QR Code 可自動帶入公司名稱、聯絡人、電話）；後台提供廠商名冊管理及績效評核 |

### 4.5 四大防護計畫

| 系統 | 路徑 | 對應法規 | 主要功能 |
|------|------|---------|---------|

### 7.3 常用維護指令

**手動觸發新聞更新（本機）**
```bash
python news/fetch_news.py
```

**重新部署 Cloudflare Worker（修改 AI 邏輯後必做）**
```bash
npx wrangler deploy workers/law-chatbot.js --name ohs-law-chatbot
```

**重新註冊週報排程（工作排程器消失時）**
```powershell
schtasks /create /tn "OHS-Portal-WeeklyReport" /tr "C:\Python314\python.exe ""C:\Users\gloom.lai\OHS-Portal\send_weekly_report.py""" /sc WEEKLY /d MON /st 09:00 /f
```

**推送網站更新（任意內容修改後）**
```bash
git add .
git commit -m "content: 說明本次修改內容"
git push
```

### 7.4 服務依賴說明

系統正常運作依賴下列外部服務，若功能異常請先確認以下項目：

- **GitHub Pages**：確認 repository 設定 → Pages 已啟用，branch 指向 `main`
- **Cloudflare Worker**：登入 `df.hr.openai@df-recycle.com` 帳號，確認 Worker `ohs-law-chatbot` 狀態正常
- **Google Apps Script**：各系統後端 Web App 需以「執行身分：我、存取權：所有人」部署，URL 填入前端 `const API`
- **IT API Gateway**：聯繫 IT 部門確認 Gemini 2.5 Flash 服務可用，Vision（圖片辨識）功能需額外確認開通
