系統概覽

此腳本可將您從公司內部工傷事故系統匯出的 Excel 檔，自動比對去重後上傳至 OHS Portal 職業災害分析系統的 Google Sheets 資料庫。
已存在的記錄不會重複新增，只會寫入新資料。

事前準備 只需做一次
1
確認 Python 已安裝
開啟 PowerShell 或終端機，輸入下方指令。若顯示版本號（如 Python 3.x.x）即代表已安裝。
python --version
⚠️ 若顯示「不是可辨識的命令」，請至 python.org 下載安裝 Python 3，安裝時勾選「Add to PATH」。
2
安裝必要套件
在終端機執行以下指令，安裝 Excel 讀取與網路請求所需的套件。
pip install pandas openpyxl requests
✅ 安裝完成後可用以下指令驗證，印出 OK 表示成功：
python -c "import pandas, openpyxl, requests; print('OK')"
3
更新 Google Apps Script（職災系統後端）
此步驟讓後端開放接收新增資料的功能。
  1. 開啟職災系統的 Google 試算表
  2. 點選上方選單 擴充功能 → Apps Script
  3. 將 accident/apps-script.gs 的全部內容貼上覆蓋現有程式碼
  4. 按 儲存（Ctrl+S）
  5. 點選 部署 → 管理部署 → 編輯（鉛筆圖示）→ 版本選「建立新版本」→ 部署
ℹ️ 部署網址不會改變，前端 index.html 不需要修改。
每次使用流程 每次匯入
1
從公司系統匯出 Excel
從公司內部工傷事故系統將資料匯出為 .xlsx 格式。 檔名不限，腳本會自動抓取資料夾中最新的 Excel 檔。
2
將 Excel 放入指定資料夾
將匯出的 Excel 檔存放至以下路徑：
C:\Users\gloom.lai\Documents\職安\職業災害分析\
ℹ️ 資料夾中可以有多個 Excel，腳本會自動選取修改日期最新的那份。
3
開啟終端機，執行匯入腳本
在 PowerShell 或終端機中輸入以下指令並按 Enter：
python C:\Users\gloom.lai\OHS-Portal\accident\import_accidents.py
4
確認預覽清單，輸入 y 新增
腳本會自動比對現有資料，列出尚未存在的新記錄，等待您確認後才寫入：
📂 讀取：Workbook2026-06-09.xlsx 共 3 筆 🔍 取得系統現有資料… 已存在（略過） 0 筆 📋 待新增 3 筆： • 2026-05-06 張秀妃 其他及無法歸類者 [五股分選廠] • 2026-05-13 蘇密 被切，割，挫傷 [全興場造粒課] • 2026-05-22 魏嘉葦 其他及無法歸類者 [矽品二林廠] 確定要新增以上 3 筆到 Google Sheets？(y/N)
確認無誤後輸入 y 並按 Enter，即可完成上傳。
✅ 成功後前往 https://df-ehs.github.io/OHS-Portal/accident/ 重新整理頁面，即可看到新增的資料。
進階用法
指令 說明
python import_accidents.py 自動抓取資料夾中最新的 Excel 執行匯入（最常用）
python import_accidents.py --dry-run 預覽模式：列出待新增清單，不實際寫入，可用於確認資料是否正確
python import_accidents.py "C:\...\檔案名稱.xlsx" 指定特定 Excel 檔案路徑，不使用自動偵測

常見問題
?
執行後顯示「❌ 錯誤：你沒有呼叫 UrlFetchApp 的權限」
Apps Script 尚未重新授權。請至 Apps Script 編輯器新增以下測試函式，執行一次後刪除，再重新部署：
function _testFetch(){ UrlFetchApp.fetch('https://www.google.com'); }
?
腳本顯示「找不到 Excel 檔案」
請確認 Excel 已放在以下資料夾，且副檔名為 .xlsx（不可為 .xls 或 .csv）：
C:\Users\gloom.lai\Documents\職安\職業災害分析\
?
資料全部顯示「已存在（略過）」，沒有新增任何記錄
腳本以「發生日期 + 員工工號」作為判斷依據。若 Excel 中的資料與系統已有記錄完全相符，即視為重複而略過。 確認 Excel 是否包含新的事故記錄，或改以 --dry-run 模式確認比對結果。
?
上傳成功但系統頁面看不到新資料
請在瀏覽器按 Ctrl + Shift + R（強制重新整理，清除快取），確保頁面讀取最新資料。
