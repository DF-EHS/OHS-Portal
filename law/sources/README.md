# 季度法規來源資料夾使用說明

## 資料夾結構

```
law/sources/
├── README.md          ← 本說明
├── 2026-Q2/           ← 已完成
└── 2026-Q3/           ← 下季度在這裡新增
    └── （下載的法規 .txt 或 .pdf 放這裡）
```

## 每季操作步驟（約 10 分鐘）

### 步驟 1：建立本季資料夾

命名規則：`YYYY-QN`（例如 `2026-Q3`）

| 季度 | 資料夾名稱 | 建議執行時間 |
|------|-----------|------------|
| 第一季 | `YYYY-Q1` | 3 月底 |
| 第二季 | `YYYY-Q2` | 6 月底 |
| 第三季 | `YYYY-Q3` | 9 月底 |
| 第四季 | `YYYY-Q4` | 12 月底 |

### 步驟 2：下載法規最新版本

前往 [全國法規資料庫](https://law.moj.gov.tw)，搜尋需要確認的法規，
點選「法規內容」→「下載」→ 選 **PDF 格式**，存入本季資料夾。

檔名請直接使用法規全名，例如：
- `職業安全衛生法.pdf`
- `職業安全衛生設施規則.pdf`
- `勞動基準法.pdf`

> PDF 會在 Agent 執行時自動轉為文字（`pdf_extract.py`），無需手動轉換。

> 只需下載「本季有修正通知」或「想重新確認」的法規，未變動的不用重複下載。

### 步驟 3：觸發 Agent 更新

在 `C:\Users\gloom.lai\OHS-Portal` 目錄下執行：

```powershell
.\law\run_quarterly_update.ps1 -Quarter 2026-Q3
```

Agent 完成後會顯示更新摘要，包含哪些條文有異動、需要人工複核。

### 步驟 4：人工複核 + 推送

1. 在 `law/index.html` 確認顯示正常
2. 找出標記為「未標記」的條文，填寫現況說明
3. 確認無誤後執行：`git push origin main`

---

## 注意事項

- 每季建立**新資料夾**，不要覆蓋舊資料（保留歷史紀錄）
- 檔名需與系統中的法規名稱相符（允許部分比對）
- 無法對應的檔案會被略過，並在摘要中列出
- 每次執行會自動備份 `index.html` 至 `law/backups/`
